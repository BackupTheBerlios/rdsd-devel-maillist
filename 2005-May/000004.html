> I have another

> decoder you are dont do any error correction.  Will
> that be done in the driver you are working on?  or is
> the error correction done on the hardware?

As for the SAA6588, error correction is done in hardware. V4L2 specs say that 
in each block there are flags that tell you wether the block is OK, had 
errors that could be corrected, or has uncorrectable errors. It's up to rdsd 
to decide how to deal with errors. This is not as trivial as it seems, e.g. 
if a block containing 4 characters of radio text has errors, should the whole 
string of 64 chars be thrown away ? What else can be done ? There are 
certainly many possible strategies. And there is not only radio text...
