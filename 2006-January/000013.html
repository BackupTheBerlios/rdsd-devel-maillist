Happy new year!
Since January, 1 there is a first alpha release (0.0.1) of the Radio Data 
System daemon (rdsd), the API library librds, and the commandline tool 
rdsquery. The project homepage is

http://rdsd.berlios.de

rdsd, librds, and rdsquery allow access to RDS data received by a Video4Linux 
radio device. You need a recent kernel that has a saa6588 module available. 
At the moment, only bttv cards have a driver function for reading RDS data 
from /dev/radio. The next thing I'll do is to build this into the saa7134 
driver, too.

The daemon rdsd can handle multiple sources (radio cards) and can distribute 
decoded data via TCP/IP or Unix domain sockets to multiple applications. The 
library librds contains all the dirty client stuff, so that a simple client 
can be written with a few lines of code. It works quite stable for me, I've 
got it in everyday use now, with an old Terratec TV/Radio+ card.

If you've got a bttv card with SAA6588 RDS decoder chip on board, please 
download and test the whole stuff. Please report your experiences on the 
rdsd-devel mailing list. Feel free to ask questions and to criticize.

Cheers,
Hans
