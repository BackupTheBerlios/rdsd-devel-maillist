> different hardware. The recommended solution is to write a V4L2 driver that

wow, this is nice. I'm also a radio amateur, but this is an other sorry.

> I've got such a device (thanks again, Rolf!) and started testing it, but

Well, my hope was, that it is possible to decode the 1,1875 kHz signal, that 
is connected to the handshake lines of a normal RS232 device. So, there is no 
need to support every single RDS decoder chip, due the serial signal is the 
same behind the rds demodulator.

But I'm not sure, if it is possible to read the handshake lines with a 
'normal' (this means: without real time extensions) linux kernel. But I'm not 
really a C programmer, so it's up to others to invest time into that ;-)

Anyhow: if nobody else have interest in reading raw rds data from the RS232 
device I may wait until there is support for an USB FM receiver, because I'm 
using a Notebook.
