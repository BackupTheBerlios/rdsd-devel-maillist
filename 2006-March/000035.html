> The goal was to attach the RDS clock and data signal to two handshake
> lines of a RS232 port.
>
> Do you have plans to support such simple adapters? Many peolple I know
> still own and use such receivers and would be happy to be able to continue
> using it for receiving and decoding RDS with linux.

Hi Michael,
as there is no standard for such devices, it's hard to support all kinds of 
different hardware. The recommended solution is to write a V4L2 driver that 
creates a /dev/radio interface for that hardware. An example is the driver 
for the "FiFi-Radio" (see http://www.ov-lennestadt.de/Projekte/ukw-radio/). 
I've got such a device (thanks again, Rolf!) and started testing it, but 
didn't get very far yet as I have very little time at the moment.

If you look at the sources of the fifi-driver, you see that it is not much. If 
you include a specialized driver for a certain serial device into rdsd, it's 
probably a similar amount of code. And that code is needed by very few 
people, but increases the size of the binary, has to be documented, 
maintained etc. For these reasons, I'm against directly supporting serial 
devices in rdsd.

Hans
