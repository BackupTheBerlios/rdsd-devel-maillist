> With deubg set to 2 I see that the RDS data is being read by the saa6588

Does the data look reasonable? (different bytes/groups, amount of bytes 
per second)
Are you sure you tuned to a station that sends RDS signals? If you didn't,
or if the RDS signal is to weak, you might see bytes coming in, but they
don't form proper RDS groups, so the rdsd decoder stage will drop them.
Note that even if the audio signal sounds strong and clear, RDS signals
can be too weak.
Do you receive radio via antenna or via cable? If it's cable, it's well
possible that they don't send all RDS groups or no RDS at all.

> When I try to use th rdsd daemon, through /dev/radio0, I don't see any
> indication that it is seeing this data.

How did you test? rdsd itself doesn't generate any output. It waits for
queries from other tools, like rdsquery. 
For debugging purposes it's useful not to start rdsd in daemon mode, then
you can see some messages on the console. See man rdsd for details.

> 
> Is there anything else I need to do to get rdsd to see this apart from
