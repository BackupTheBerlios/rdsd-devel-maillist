Hi Daniele,
rather than writing your own software, I'd suggest you add the missing TMC 
features to rdsd. I'm planning to extend the API so you can request TMC data 
either in encoded form (event 123 at location 456) or in decoded form using 
strings from a (yet to be created) database. Events are the same for all 
countries, but location tables are of course country specific. 

If you have special needs/suggestions regarding the API, please let us know. 
It should be possible to extend rdsd/librds to meet your requirements.
