>
> other point : I can't find any tuner FM/TV card that has this chip
> (SAA6588 by Philips), can someone give me some URL so I can buy it ?

Well, the problem at the moment is that a standard kernel supports only RDS on 
bttv cards. I sent a patch for the saa7134 driver to Video4Linux mailing 
list, but it got lost somehow. I'll send it again as soon as I find the time.

Unfortunately, AFAIK all the cards in the market right now are SAA7134 cards, 
a few examples are given in the V4L wiki:

http://www.linuxtv.org/v4lwiki/index.php/Radio_Data_System_(RDS)

You have to decide wether you want to help getting RDS on SAA7134 to work or 
you can try to get an old bttv card, e.g. from eBay.

>
> The RSD signal is just then 57kHz band from the audio signal. May it
> be possible to extract the RSD information from any tuner FM card. If
> the card does not filter it, I can work on it (signal processing with
> Scilab first, C++ program then), so that it could be accesible from
> any normal card by any normal user ?

Well, if you're good enough at hardware development, you can perhaps solder an 
SAA6588 to the audio output of the tuner, and use the video chip's I2C bus to 
connect it. The SAA6588 will be found out of the box by the kernel. Problem 
could be that not all tuners have audio outputs, I think some only output IF 
signals, and audio demodulation takes place inside the chip. Not sure about 
this, though.
