Am Donnerstag 17 Mai 2007 schrieb Tom Laermans:
>
> Does the card logic, or rdsd (or anything else) automatically switch
> frequency (like a car radio) when the current frequency reception gets
> bad? That would be something I need...

Not automatically. librds has a function to query signal strength and
another to set the frequency, so a userspace application could do that.

However, it's not trivial to define what "signal quality" is. For 
listening to radio, a value proportional to RF field strength might be
appropriate. For RDS data reception, there's a certain minimal level for
the decoder to work. If you're near that level, the number of error blocks
rapidly increases until no reasonable reception is possible anymore.
It is well possible that you have the impression of a strong station
(because audio is loud and clear), but RDS is hardly working.

Another thing you'll need is the alternative frequency list for a station.
Code for that is not complete in rdsd, there's still some work to do.
