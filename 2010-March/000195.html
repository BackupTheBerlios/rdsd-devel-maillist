Well, my (incomplete) list would include things like:

- alternative frequency list decoding (needed for all kinds of mobile
  applications).
- TMC decoding. Event codes are simple since they're the same for each country,
  but location codes are country specific. Some countries have more than one
  location code table. Decoding is not really simple, e.g. there can be "hops"
  over several (contiguous) locations on one road. A decent API needs to be
  defined and implemented, so that someone could write an application like
  show-traffic-jams-on-a-map.
- The RDS protocol contains information about the TMC location table. Decode
  this and choose the appropriate location file to return locations as strings.
  If we don't have such a file (they're not free in most countries), return the
  location number.
