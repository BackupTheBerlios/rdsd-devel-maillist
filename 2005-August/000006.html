This is a copy of a mail I just sent to the video4linux list:

Hi,
as I recently posted, I've got a kernel module for acquiring RDS data from a 
SAA6588. The card driver has to be changed to enable reads from /dev/radio. I 
only tested that with bttv so far, the changes are fairly simple, it should 
be possible to apply this to other drivers as well.

I attach the bttv-driver.c patch to receive your opinions. The patch has two 
purposes:
1) Allow multiple open() calls to /dev/radio
2) Add read functionality for /dev/radio

The sources for saa6588.ko (and other stuff) are available on the rdsd 
homepage http://rdsd.berlios.de

Today I added a small "Hello RDS world" test program (read_rds.c). It reads 
data from /dev/radio and decodes some blocks (radiotext, program name and 
type, date/time). It's quite useful for testing.

Greetings,
Hans
